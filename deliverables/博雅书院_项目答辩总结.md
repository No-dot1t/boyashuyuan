## 一、项目概述

### 1.1 项目背景

博雅书院是一个面向高校师生群体的综合性智慧校园平台。平台以"知识管理 + 学习成长 + 校园生活"为三大核心维度，构建了一个集**数字图书馆、在线课程、智能推荐、自习管理、校园元宇宙、学术讲座、校友网络、内容审核**等功能于一体的全栈 Web 应用系统。

### 1.2 项目定位

| 维度 | 描述 |
|------|------|
| **用户群体** | 高校学生、教师、管理员 |
| **核心价值** | 提供一站式学习管理、知识发现与校园社交体验 |
| **技术特色** | 纯原生 Java Web 架构、AI 驱动推荐、科技风 UI 设计 |
| **部署方式** | WAR 包部署于 Tomcat 7，端口 8081 |

### 1.3 项目规模

| 指标 | 数量 |
|------|------|
| Java 源文件 | **123 个**（含 57 个 Servlet、24 个 DAO、27 个 POJO、3 个 Service、5 个 Util、2 个 Filter） |
| JSP 页面 | **39 个**（含主框架 + 33 个功能子页面 + 3 个管理列表页 + 2 个登录页） |
| CSS 样式文件 | **24 个**（含 8 套主题系统 + 子页面浅色适配层） |
| JS 脚本文件 | **3 个**（核心框架 3,221 行 + ECharts 图表库 98K 行） |
| 数据库表 | **64 张**（核心业务表，含完整种子数据） |
| SQL 脚本 | **1 个**（1,604 行建表 + 种子数据） |
| 代码总行数 | 约 **70,000+ 行**（不含第三方库） |
| Maven 依赖 | **10 个**第三方库 |

### 3.2 设计模式与原则

| 模式/原则 | 应用场景 |
|-----------|----------|
| **MVC 三层架构** | Servlet(Controller) + JSP(View) + DAO/POJO(Model) |
| **单例模式** | DBManager 连接池、NotificationScheduler 定时器 |
| **DAO 模式** | 每张核心表对应一个 DAO 类，封装数据访问 |
| **POJO 映射** | 每张表对应一个 POJO 实体类 |
| **注解式路由** | @WebServlet + @WebFilter + @WebListener |
| **参数分发模式** | 通过 `action` 参数在同 Servlet 中分流操作 |
| **观察者模式** | ServletContextListener 启动/销毁监听 |

### 3.3 请求生命周期

```
浏览器请求 → GlobalExceptionFilter（全局异常捕获）
           → Servlet（解析 action 参数）
           → DAO（执行 SQL 查询）
           → DBManager.getConnection()（从 HikariCP 获取连接）
           → MySQL（执行查询）
           → DAO（封装为 POJO）
           → Servlet（JSON 序列化 / JSP 转发）
           → 浏览器（渲染响应）
```

### 4.4 数据库设计亮点

1. **外键完整性**：核心表之间外键约束完善，`ON DELETE CASCADE/SET NULL` 策略合理
2. **唯一约束覆盖**：所有关联表均设置 `UNIQUE KEY` 防止数据重复
3. **软删除设计**：`private_message` 实现发送者/接收者独立软删除
4. **多态评价**：`user_evaluations` 通过 `target_type` 字段支持课程/讲座/图书多目标评价
5. **AI 集成**：`content_review` 表含 `ai_score`、`ai_feedback` 字段支持 AI 辅助审核
6. **时间追踪**：核心表均有 `created_at`/`updated_at` 自动时间戳

**文件**：`LoginServlet.java`、`RegisterServlet.java`、`ForgotPasswordServlet.java`

```
功能流程：
 注册 → BCrypt密码加密 → 存入users表
 登录 → BCrypt.verify()密码校验 → Session存储currentUser
 权限 → 通过session中user.role字段（admin/user）控制菜单显示

- 密码使用 BCrypt 加盐哈希，不可逆存储

- JSP 层通过 `<% if (isAdmin) { %>` 控制管理菜单渲染
- CSRF Token 每会话生成，管理表单提交验证

### 5.3 智能推荐引擎

**文件**：`RecommendEngine.java`（690行）、`RecommendServlet.java`

**核心算法**：

| 算法组件 | 描述 |
|----------|------|
| **用户画像构建** | 综合 5 个维度数据源（浏览/收藏/下载/阅读/场景访问），多维度加权提取兴趣标签 |
| **行为权重** | 收藏×5.0 > 评分≥4×4.0 > 下载×3.0 > 阅读×2.0 > 场景访问×1.5 > 浏览×1.0 |
| **时间衰减** | 30天衰减函数 `max(0.1, 1.0 - days/30)`，越近行为权重越高 |
| **中文关键词提取** | 内置 30+ 学科关键词库，简易分词匹配 |
| **协同过滤** | 通过 `user_bookmark` 找到行为相似用户，推荐交叉内容 |
| **内容匹配** | 兴趣标签 LIKE 匹配课程/图书/讲座的名称、分类、描述 |
| **推荐理由** | 每项推荐附带"根据你对「XX」的兴趣推荐"解释 |

```
推荐流程：
 用户请求 → buildUserProfile() 画像构建
          → recommendBooks()   图书匹配（标签LIKE书名/分类/简介）
          → recommendCourses() 课程匹配（综合评分×0.6+报名数×0.4排序）
          → recommendLectures() 讲座匹配（按日期+观看量排序）
          → collaborativeFilter() 协同过滤补充
          → 热门后备（当无匹配时降级）
```

│    │     ├─ addBookmarkTags()  收藏的书           │
│    │     ├─ addHistoryTags()   历史记录           │
│    │     ├─ addRatingTags()    高分评价           │
│    │     └─ addSceneTags()     场景访问           │
│    │     ↓ 权重×时间衰减 → 归一化到0-100          │
│    │                                              │
│    ├─ 2. getKnownIds(userId)   已接触内容（排除）  │
│    │                                              │
│    ├─ 3. 内容匹配推荐                             │
│    │     ├─ recommendBooks()   标签LIKE书名/分类   │
│    │     ├─ recommendCourses() 标签LIKE课程名/描述 │
│    │     └─ recommendLectures() 标签LIKE讲座标题   │
│    │                                              │
│    ├─ 4. collaborativeFilter() 协同过滤补充       │
│    │     └─ 相似用户收藏交叉推荐                   │
│    │                                              │
│    └─ 5. 热门后备（无匹配时降级为热门推荐）        │
└──────────────────────────────────────────────────┘
```

**行为权重矩阵**：
```
收藏(bookmark)     ████████████████████  ×5.0
高分评分(rating≥4) ████████████████      ×4.0
下载(download)      ████████████          ×3.0
阅读(read)          ████████              ×2.0
场景访问(scene)     ██████                ×1.5
浏览(view)          ████                  ×1.0
                          × timeDecay(30天)
```

### 11.1 项目总结

博雅书院项目是一个**全栈自研**的综合性智慧校园平台，从数据库设计、后端 Servlet 开发、前端 JSP 渲染到 AI 智能推荐引擎、全格式电子书阅读器，实现了**端到端的完整覆盖**。

- 具备全栈开发能力（数据库 → 后端 → 前端 → AI 集成）
